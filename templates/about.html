多用户隔离 文件夹管理 上传进度 CSRF 防护 移动端适配 密码哈希

功能特性

多用户支持
  • 首次初始化：系统首次启动时强制创建唯一管理员账户
  • 管理员面板：管理员可在 Web 界面中创建普通用户并查看用户列表
  • 用户隔离：每个用户的上传文件、文件夹均通过 user_id 进行数据库级与逻辑隔离
文件管理
  • 文件夹操作：支持创建子文件夹，面包屑导航跳转
  • 递归删除：管理员可删除文件夹及其所有子内容
  • 文件名截断：超过 25 字符显示省略号，悬停可见全名
  • 文件上传：支持任意类型，前端体积校验 + 实时进度条
  • 文件下载：通过动态路由提供下载，校验归属权限
  • 存储方式：文件以时间戳重命名，保存在 uploads/<user_id>/ 目录
安全机制
  • 密码哈希：所有用户密码通过 werkzeug.security 加密存储，不可逆
  • CSRF 防护：全局生成并校验 CSRF Token，防止跨站请求伪造
  • 登录保护：所有文件管理路由均使用 login_required 装饰器验证会话
  • 管理员专属：/admin 路由被 admin_required 装饰器保护
  • 初始化钩子：无用户时强制重定向至 /init 页面
  • 路径穿越防护：is_safe_path() 函数防止访问 uploads/ 以外的文件
界面
  • 响应式布局：基于 Bootstrap 5，深蓝导航栏，Vanta.js 动态背景
  • 面包屑导航：方便浏览多级目录
  • 移动端适配：表格横向滑动、汉堡菜单折叠导航、文件名截断显示

快速开始

适用于本地开发 / 测试 · 要求 Python 3.7+ 及 pip

  1. 获取代码：克隆或解压项目源码到本地
  2. 创建虚拟环境（推荐）： 
    python -m venv venv
source venv/bin/activate   # Linux/Mac
venv\Scripts\activate      # Windows
  3. 安装依赖（（requirements.txt 内容：Flask、、Werkzeug、、flask-compress、、gunicorn）： 
    pip install -r requirements.txt
  4. 检查配置：config.json 中 max_file_size_mb 默认值为 100，可按需修改
  5. 清理旧数据（如有需要）：删除 uploads.db 即可重新初始化管理员
  6. 启动应用： 
    python server.py
    服务将在 http://127.0.0.1:5000 运行
  7. 初始化管理员：浏览器访问上述地址，系统会自动跳转到 /init 页面完成注册并自动登录

使用指南

初始化管理员
  • 首次访问系统（无任何用户）时，任何路径都会被重定向至 /init
  • 输入管理员用户名、密码及确认密码
  • 用户名仅允许 3~20 位字母、数字或下划线；密码至少 6 位
  • 创建成功后自动登录并跳转到文件管理页面
管理员操作
  • 创建普通用户：登录后点击"管理用户"进入 /admin 面板，填写新用户信息即可创建
  • 文件管理：与普通用户界面相同，可上传、下载、创建文件夹，数据同样隔离存储
普通用户操作
  • 使用管理员创建的账号在 /login 页面登录
  • 浏览文件：登录后进入根目录，通过面包屑导航进入子文件夹
  • 创建文件夹：填写文件夹名（不能包含 / \ : * ? " < > | 等字符），点击创建
  • 上传文件：选择文件 → 点击"上传"，前端校验体积并显示进度条
  • 下载文件：点击文件列表中的文件名即可下载原始文件
  • 移动端使用：文件名超过 25 字符会自动截断并显示 "..."，可横向滑动表格；手机端点击汉堡按钮展开菜单

部署与安全

Gunicorn 启动（生产环境推荐） 
# 安装
pip install gunicorn flask-compress

# 启动（4 worker）
gunicorn -w 4 -b 127.0.0.1:5000 server:app

# 或使用项目提供的 start.sh 脚本
./start.sh

应用已自动启用 Gzip 压缩和代理头修正（（ProxyFix），无需额外配置。

Nginx 反向代理配置 
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 100m;   # 需与 config.json 中 max_file_size_mb 一致

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 禁止直接访问 uploads 目录（所有下载均通过 /download 路由提供）
    location /uploads {
        deny all;
    }
}
HTTPS 配置（推荐）

使用 Let's Encrypt 获取证书，并配置 Nginx 监听 443 端口。同时在 server.py 中设置：

app.config.update(
    SESSION_COOKIE_SECURE=True,
    SESSION_COOKIE_HTTPONLY=True,
    SESSION_COOKIE_SAMESITE='Lax',
)
进程管理

可使用 systemd 或 supervisor 管理 Gunicorn 服务，保证后台运行及开机自启。

安全注意事项
密钥管理：应用首次启动会自动生成固定密钥到 secret_config.py。生产环境建议通过环境变量自定义：export SECRET_KEY="your-generated-secret-key"
关闭调试模式：切勿在生产环境开启 Flask 的 debug=True
限制上传目录访问：通过 Nginx 配置禁止直接访问 /uploads 路径，所有文件获取必须经过应用鉴权
保护数据库：uploads.db 不要直接暴露在 Web 可访问目录下
定期备份：定期复制 uploads/ 文件夹和 uploads.db 文件以保证数据安全
CSRF 保护：所有 POST 请求（包括删除操作）均验证 CSRF Token
文件类型白名单（可扩展）：当前未限制文件类型，若需提高安全性，可在上传逻辑中增加 MIME 或扩展名白名单校验

常见问题

Q：首次访问没有跳转到初始化页面，而是直接显示登录页？

请检查是否存在 uploads.db 并且其中已有用户数据（例如之前的残留数据）。删除该文件及 uploads/ 目录后重启服务即可。

Q：上传大文件时失败，提示文件过大？

首先确认 config.json 中的 max_file_size_mb；如果使用了 Nginx 反向代理，还需确保 client_max_body_size 的值不小于该配置。

Q：管理员忘记密码怎么办？

目前没有内置密码重置功能。可通过 Python 交互环境手动重置：

from werkzeug.security import generate_password_hash
import sqlite3

new_hash = generate_password_hash('新密码')
conn = sqlite3.connect('uploads.db')
conn.execute("UPDATE users SET password_hash = ? WHERE id = ?", (new_hash, 管理员ID))
conn.commit()
conn.close()
Q：如何迁移或备份数据？

直接备份 uploads/ 目录（所有文件）和 uploads.db（数据库记录）。恢复时将两者放回原位置即可。

Q：能否限制用户存储空间？

当前版本未实现配额管理，所有用户共享同一个磁盘空间（实际物理磁盘大小限制）。如有需求可参考"未来扩展方向"进行二次开发。

Q：手机访问时导航栏按钮消失？

当屏幕宽度小于 992px 时，导航栏会自动折叠为汉堡菜单，点击即可展开。

Q：文件名显示不全或被截断？

移动端文件名超过 25 字符会自动截断并显示 "..."，可横向滑动表格查看完整名称，鼠标悬停也有提示。
